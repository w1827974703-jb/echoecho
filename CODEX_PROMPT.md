# 给 Codex 的提示词

> 复制下面「---」之间的内容发给 Codex 即可。

---

你好，请接手一个已完成核心功能的 Next.js 项目，帮我把**前端跑起来**并做后续前端开发。

**项目**：EchoEcho —— 听力优先的英文播客精听 Web 工具（上传英文音频 → 转录带时间戳 → 边听边按需看字幕 → 点词看释义并记录 → AI 出题/短文复习）。

**第一步（务必先做）**：完整阅读仓库根目录的 `HANDOFF_FOR_CODEX.md`，它讲清了技术架构、前后端契约、功能清单（具体到文件）、以及跑起来的坑。再快速扫一眼 `CLAUDE.md`（工程规范，最权威）和 `AGENTS.md`。

**几条必须遵守的硬约束**：
1. **这不是你训练数据里的 Next.js**——用的是 **Next.js 16（App Router）+ React 19 + Tailwind v4 + shadcn/ui**，有破坏性变更。写代码前先查 `node_modules/next/dist/docs/`，别凭记忆用旧 API。
2. **安全红线（最高优先级）**：所有 AI/ASR 调用只在服务端 `src/app/api/*` 里发起；密钥（`DASHSCOPE_API_KEY`、`OSS_*`）只存 `.env.local`，**绝不能出现在任何前端代码、客户端组件、或发往浏览器的响应里**。前端只跟自己的 `/api/*` 通信。
3. 源码根是 **`src/`**（不是 `app/`）；内页在 `src/app/(app)/` 路由组下，共享左侧边栏布局；封面页 `src/app/page.tsx` 是全屏、独立的。
4. **没有数据库、没有账号系统**：用户数据全在浏览器本地（localStorage 存元数据 / IndexedDB 存音频 / sessionStorage 存巩固结果）。
5. **时间戳统一「秒」**（Paraformer 返回毫秒，服务端已 /1000）。

**跑起来的现实**：
- `npm install && npm run dev`，开 `http://localhost:3000`。
- 页面 UI 都能打开，但**转录/查词/出题/短文**需要 `.env.local` 里的阿里云百炼 Key 和 OSS 凭证才能真正工作。如果我没给你 Key，请：**先把纯前端 UI 跑起来**，AI/转录相关功能允许失败或先 mock（`HANDOFF_FOR_CODEX.md` §6.4 讲了怎么造本地样例数据、以及哪 5 个 `/api/*` 需要 mock）。别为了让它「不报错」而把密钥硬编码或搬到前端。

**当前手上的任务**（除非我另说）：先确认前端能完整跑起来、各页面渲染正常；有报错先定位是「缺环境变量」还是「真 bug」，分类告诉我。

**注意**：如果看到根目录有 `内页配色示意.html`，那是配色示意稿、不是产品代码，别当成页面去改。正在进行的是内页视觉优化（只动 `src/app/globals.css` 的 token + 少量组件 class，不改功能、不改布局）——这块等我确认方向后再说，你先别动样式。

有任何不确定的地方（尤其涉及改动功能逻辑或依赖），先问我再动手。
